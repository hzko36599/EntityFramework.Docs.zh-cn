## <a name="entity-states-and-savechanges"></a><span data-ttu-id="f0f8b-107">实体状态和 SaveChanges</span><span class="sxs-lookup"><span data-stu-id="f0f8b-107">Entity states and SaveChanges</span></span>

<span data-ttu-id="f0f8b-108">实体可以是 EntityState 枚举定义的五种状态之一。</span><span class="sxs-lookup"><span data-stu-id="f0f8b-108">An entity can be in one of five states as defined by the EntityState enumeration.</span></span> <span data-ttu-id="f0f8b-109">这些状态包括：</span><span class="sxs-lookup"><span data-stu-id="f0f8b-109">These states are:</span></span>

- <span data-ttu-id="f0f8b-111">保持不变：上下文正在跟踪实体，该实体存在于数据库中，并且其属性值未更改为数据库中的值</span><span class="sxs-lookup"><span data-stu-id="f0f8b-111">Unchanged: the entity is being tracked by the context and exists in the database, and its property values have not changed from the values in the database</span></span>  
- <span data-ttu-id="f0f8b-112">已修改：实体正在由上下文跟踪，并存在于数据库中，并且其部分或全部属性值已修改</span><span class="sxs-lookup"><span data-stu-id="f0f8b-112">Modified: the entity is being tracked by the context and exists in the database, and some or all of its property values have been modified</span></span>

- <span data-ttu-id="f0f8b-120">删除的实体将从数据库中删除，然后与上下文分离。</span><span class="sxs-lookup"><span data-stu-id="f0f8b-120">Deleted entities are deleted from the database and are then detached from the context.</span></span>  

<span data-ttu-id="f0f8b-121">下面的示例演示了如何更改实体或实体关系图的状态。</span><span class="sxs-lookup"><span data-stu-id="f0f8b-121">The following examples show ways in which the state of an entity or an entity graph can be changed.</span></span>  

## <a name="adding-a-new-entity-to-the-context"></a><span data-ttu-id="f0f8b-122">向上下文中添加新实体</span><span class="sxs-lookup"><span data-stu-id="f0f8b-122">Adding a new entity to the context</span></span>  

<span data-ttu-id="f0f8b-123">可以通过对 DbSet 调用 Add 方法，将新实体添加到上下文中。</span><span class="sxs-lookup"><span data-stu-id="f0f8b-123">A new entity can be added to the context by calling the Add method on DbSet.</span></span>
<span data-ttu-id="f0f8b-124">这会使实体处于已添加状态，这意味着它将在下一次调用 SaveChanges 时插入到数据库中。</span><span class="sxs-lookup"><span data-stu-id="f0f8b-124">This puts the entity into the Added state, meaning that it will be inserted into the database the next time that SaveChanges is called.</span></span>
<span data-ttu-id="f0f8b-125">例如：</span><span class="sxs-lookup"><span data-stu-id="f0f8b-125">For example:</span></span>  

``` csharp
using (var context = new BloggingContext())
{
    var blog = new Blog { Name = "ADO.NET Blog" };
    context.Blogs.Add(blog);
    context.SaveChanges();
}
```  

<span data-ttu-id="f0f8b-126">向上下文中添加新实体的另一种方法是将其状态更改为 "已添加"。</span><span class="sxs-lookup"><span data-stu-id="f0f8b-126">Another way to add a new entity to the context is to change its state to Added.</span></span> <span data-ttu-id="f0f8b-127">例如：</span><span class="sxs-lookup"><span data-stu-id="f0f8b-127">For example:</span></span>  

``` csharp
using (var context = new BloggingContext())
{
    var blog = new Blog { Name = "ADO.NET Blog" };
    context.Entry(blog).State = EntityState.Added;
    context.SaveChanges();
}
```  

<span data-ttu-id="f0f8b-128">最后，可以通过将新实体挂钩到已跟踪的另一个实体来向上下文中添加新实体。</span><span class="sxs-lookup"><span data-stu-id="f0f8b-128">Finally, you can add a new entity to the context by hooking it up to another entity that is already being tracked.</span></span>
<span data-ttu-id="f0f8b-129">这可以是将新实体添加到另一个实体的集合导航属性中，或通过设置另一个实体的引用导航属性来指向新实体。</span><span class="sxs-lookup"><span data-stu-id="f0f8b-129">This could be by adding the new entity to the collection navigation property of another entity or by setting a reference navigation property of another entity to point to the new entity.</span></span> <span data-ttu-id="f0f8b-130">例如：</span><span class="sxs-lookup"><span data-stu-id="f0f8b-130">For example:</span></span>  

``` csharp
using (var context = new BloggingContext())
{
    // Add a new User by setting a reference from a tracked Blog
    var blog = context.Blogs.Find(1);
    blog.Owner = new User { UserName = "johndoe1987" };

    // Add a new Post by adding to the collection of a tracked Blog
    blog.Posts.Add(new Post { Name = "How to Add Entities" });

    context.SaveChanges();
}
```  

<span data-ttu-id="f0f8b-131">请注意，对于所有这些示例，如果添加的实体具有对尚未跟踪的其他实体的引用，则这些新实体也将添加到上下文中，并将在下次调用 SaveChanges 时插入到数据库中。</span><span class="sxs-lookup"><span data-stu-id="f0f8b-131">Note that for all of these examples if the entity being added has references to other entities that are not yet tracked then these new entities will also be added to the context and will be inserted into the database the next time that SaveChanges is called.</span></span>  

## <a name="attaching-an-existing-entity-to-the-context"></a><span data-ttu-id="f0f8b-132">将现有实体附加到上下文</span><span class="sxs-lookup"><span data-stu-id="f0f8b-132">Attaching an existing entity to the context</span></span>  

<span data-ttu-id="f0f8b-133">如果你的实体已存在于数据库中，但当前未由上下文跟踪，则可以使用 DbSet 上的 Attach 方法告诉上下文跟踪实体。</span><span class="sxs-lookup"><span data-stu-id="f0f8b-133">If you have an entity that you know already exists in the database but which is not currently being tracked by the context then you can tell the context to track the entity using the Attach method on DbSet.</span></span> <span data-ttu-id="f0f8b-134">实体在上下文中将处于未更改状态。</span><span class="sxs-lookup"><span data-stu-id="f0f8b-134">The entity will be in the Unchanged state in the context.</span></span> <span data-ttu-id="f0f8b-135">例如：</span><span class="sxs-lookup"><span data-stu-id="f0f8b-135">For example:</span></span>  

``` csharp

    context.Blogs.Attach(existingBlog);

    // Do some more work...  

    context.SaveChanges();
}
```  

<span data-ttu-id="f0f8b-136">请注意，如果在调用 SaveChanges 时未执行附加实体的任何其他操作，则不会对数据库进行任何更改。</span><span class="sxs-lookup"><span data-stu-id="f0f8b-136">Note that no changes will be made to the database if SaveChanges is called without doing any other manipulation of the attached entity.</span></span> <span data-ttu-id="f0f8b-137">这是因为实体处于未更改状态。</span><span class="sxs-lookup"><span data-stu-id="f0f8b-137">This is because the entity is in the Unchanged state.</span></span>  

<span data-ttu-id="f0f8b-138">将现有实体附加到上下文的另一种方法是将其状态更改为 "未更改"。</span><span class="sxs-lookup"><span data-stu-id="f0f8b-138">Another way to attach an existing entity to the context is to change its state to Unchanged.</span></span> <span data-ttu-id="f0f8b-139">例如：</span><span class="sxs-lookup"><span data-stu-id="f0f8b-139">For example:</span></span>  

``` csharp
var existingBlog = new Blog { BlogId = 1, Name = "ADO.NET Blog" };

using (var context = new BloggingContext())
{
    context.Entry(existingBlog).State = EntityState.Unchanged;

    // Do some more work...  

    context.SaveChanges();
}
```  

<span data-ttu-id="f0f8b-140">请注意，对于这两个示例，如果附加的实体已引用尚未跟踪的其他实体，则这些新实体还会附加到处于未更改状态的上下文。</span><span class="sxs-lookup"><span data-stu-id="f0f8b-140">Note that for both of these examples if the entity being attached has references to other entities that are not yet tracked then these new entities will also attached to the context in the Unchanged state.</span></span>  

## <a name="attaching-an-existing-but-modified-entity-to-the-context"></a><span data-ttu-id="f0f8b-141">将现有的但修改的实体附加到上下文</span><span class="sxs-lookup"><span data-stu-id="f0f8b-141">Attaching an existing but modified entity to the context</span></span>  

<span data-ttu-id="f0f8b-142">如果你的实体已存在于数据库中，但可能已对其进行了更改，则可以通知上下文附加实体并将其状态设置为 "已修改"。</span><span class="sxs-lookup"><span data-stu-id="f0f8b-142">If you have an entity that you know already exists in the database but to which changes may have been made then you can tell the context to attach the entity and set its state to Modified.</span></span>

``` csharp

using (var context = new BloggingContext())
{
    context.Entry(existingBlog).State = EntityState.Modified;

    // Do some more work...  

    context.SaveChanges();
}
```  

<span data-ttu-id="f0f8b-144">当你将状态更改为 "已修改" 时，实体的所有属性都将标记为已修改，并且在调用 SaveChanges 时，所有属性值都将发送到数据库。</span><span class="sxs-lookup"><span data-stu-id="f0f8b-144">When you change the state to Modified all the properties of the entity will be marked as modified and all the property values will be sent to the database when SaveChanges is called.</span></span>  

<span data-ttu-id="f0f8b-145">请注意，如果附加的实体具有对尚未跟踪的其他实体的引用，则这些新实体将以未更改状态附加到上下文中，而不会自动进行修改。</span><span class="sxs-lookup"><span data-stu-id="f0f8b-145">Note that if the entity being attached has references to other entities that are not yet tracked, then these new entities will attached to the context in the Unchanged state—they will not automatically be made Modified.</span></span>
<span data-ttu-id="f0f8b-146">如果需要将多个实体标记为 "已修改"，则应单独设置每个实体的状态。</span><span class="sxs-lookup"><span data-stu-id="f0f8b-146">If you have multiple entities that need to be marked Modified you should set the state for each of these entities individually.</span></span>  

## <a name="changing-the-state-of-a-tracked-entity"></a><span data-ttu-id="f0f8b-147">更改所跟踪实体的状态</span><span class="sxs-lookup"><span data-stu-id="f0f8b-147">Changing the state of a tracked entity</span></span>  

<span data-ttu-id="f0f8b-148">您可以通过对其项设置状态属性来更改已被跟踪的实体的状态。</span><span class="sxs-lookup"><span data-stu-id="f0f8b-148">You can change the state of an entity that is already being tracked by setting the State property on its entry.</span></span> <span data-ttu-id="f0f8b-149">例如：</span><span class="sxs-lookup"><span data-stu-id="f0f8b-149">For example:</span></span>

``` csharp
    context.Blogs.Attach(existingBlog);

    // Do some more work...  

    context.SaveChanges();
}
```  

<span data-ttu-id="f0f8b-150">请注意，为已跟踪的实体调用 "添加或附加" 也可用于更改实体状态。</span><span class="sxs-lookup"><span data-stu-id="f0f8b-150">Note that calling Add or Attach for an entity that is already tracked can also be used to change the entity state.</span></span> <span data-ttu-id="f0f8b-151">例如，为当前处于添加状态的实体调用 Attach 会将其状态更改为 "不更改"。</span><span class="sxs-lookup"><span data-stu-id="f0f8b-151">For example, calling Attach for an entity that is currently in the Added state will change its state to Unchanged.</span></span>  

## <a name="insert-or-update-pattern"></a><span data-ttu-id="f0f8b-152">插入或更新模式</span><span class="sxs-lookup"><span data-stu-id="f0f8b-152">Insert or update pattern</span></span>  

<span data-ttu-id="f0f8b-153">某些应用程序的一种常见模式是将实体添加为新 (导致数据库插入) 或将实体附加为现有，并将其标记为已修改 (导致数据库更新) ，具体取决于主键的值。</span><span class="sxs-lookup"><span data-stu-id="f0f8b-153">A common pattern for some applications is to either Add an entity as new (resulting in a database insert) or Attach an entity as existing and mark it as modified (resulting in a database update) depending on the value of the primary key.</span></span>
<span data-ttu-id="f0f8b-154">例如，在使用数据库生成的整数主键时，通常将包含零键的实体作为新的和具有非零键的实体视为现有的。</span><span class="sxs-lookup"><span data-stu-id="f0f8b-154">For example, when using database generated integer primary keys it is common to treat an entity with a zero key as new and an entity with a non-zero key as existing.</span></span>
<span data-ttu-id="f0f8b-155">可以通过基于主键值的检查设置实体状态来实现此模式。</span><span class="sxs-lookup"><span data-stu-id="f0f8b-155">This pattern can be achieved by setting the entity state based on a check of the primary key value.</span></span> <span data-ttu-id="f0f8b-156">例如：</span><span class="sxs-lookup"><span data-stu-id="f0f8b-156">For example:</span></span>  

``` csharp
public void InsertOrUpdate(Blog blog)
{
    using (var context = new BloggingContext())
    {
        context.Entry(blog).State = blog.BlogId == 0 ?
                                   EntityState.Added :
                                   EntityState.Modified;

        context.SaveChanges();
    }
}
```  

<span data-ttu-id="f0f8b-157">请注意，当你将状态更改为 "已修改" 时，实体的所有属性都将标记为已修改，并且在调用 SaveChanges 时，所有属性值都将发送到数据库。</span><span class="sxs-lookup"><span data-stu-id="f0f8b-157">Note that when you change the state to Modified all the properties of the entity will be marked as modified and all the property values will be sent to the database when SaveChanges is called.</span></span>
